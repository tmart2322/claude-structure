# Phase reviews

This directory is **generated by `/phase-builder`** — you don't hand-author files here. When a phase starts, the `phase-entry` workflow commits a set of four files for that phase:

- **`phase-<n>-entry.md`** — the entry review (tenets re-read, scope check, open questions, risk delta, sharpened definition of done, build plan summary). The operator gives go/no-go on it.
- **`phase-<n>-tasks.md`** — the task ledger: one line per build chunk (`[ ]` pending · `[>]` in progress · `[x]` done · `[!]` blocked), grouped into waves. The **source of truth for "what's left"**, committed on the phase branch and resumable across sessions. Holds build chunks + `(FB-n)` operator-feedback tasks only — *not* validations or gates (those are exit gates).
- **`phase-<n>-exit.md`** — the exit review: a DRAFT skeleton at entry (gate ledger G1–G11 all ⬜, Validation table seeded), filled through the phase. The durable, resumable home for gate + validation state.
- **`phase-<n>-operator-steps.md`** — the operator's at-a-glance "what do I have to do": Before / During / Validation / After gestures, with inline `> FEEDBACK: ...` notes the next `/phase-builder` run triages.

The templates for the entry/exit/operator-steps files live in `.claude/skills/phase-builder/references/review-templates.md`. See [`../REVIEW.md`](../REVIEW.md) for what each gate means and the README for how the loop runs.

This file is just a placeholder so the directory exists in a fresh checkout.
